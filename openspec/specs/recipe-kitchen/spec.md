# 菜谱与做饭记录 — 系统规格

## 需求

### REQ-1 菜谱 CRUD

系统 **SHALL** 允许用户创建、查看、编辑与删除菜谱。

**场景 S1.1**  
**GIVEN** 用户在新建菜谱页  
**WHEN** 用户填写菜名、可选封面、食材清单（每行一项）、步骤（每行一步）并保存  
**THEN** 系统持久化该菜谱并在详情页展示。

**场景 S1.2**  
**GIVEN** 用户打开某菜谱详情  
**WHEN** 用户点击编辑并修改字段后保存  
**THEN** 系统更新该菜谱且列表与详情反映最新内容。

**场景 S1.3**  
**GIVEN** 用户在菜谱详情页  
**WHEN** 用户确认删除菜谱  
**THEN** 系统移除该菜谱及其下所有做饭记录。

### REQ-2 做饭记录

系统 **SHALL** 允许用户在指定菜谱下创建与编辑做饭记录；记录 **SHALL** 支持多张实拍图与文字说明。

**场景 S2.1**  
**GIVEN** 用户在菜谱详情页  
**WHEN** 用户新建记录并上传图片、填写日期与文字后保存  
**THEN** 系统保存记录并将该菜谱标记为「已做过」。

**场景 S2.2**  
**GIVEN** 已存在一条记录  
**WHEN** 用户编辑并保存  
**THEN** 系统更新该记录内容。

### REQ-3 做过 / 未做过

系统 **SHALL** 允许用户将菜谱标记为做过或未做过，并支持按该状态筛选列表。

**场景 S3.1**  
**GIVEN** 用户在菜谱表单或详情  
**WHEN** 用户切换「做过」状态  
**THEN** 系统在持久化层更新该标记并在筛选中生效。

### REQ-4 计划（本周 / 指定日期）

系统 **SHALL** 允许将菜谱标记为「本周计划」或绑定到未来某一日期；列表 **SHALL** 提供「本周计划」筛选。

**场景 S4.1**  
**GIVEN** 用户为菜谱选择「本周计划」  
**WHEN** 用户打开首页「本周计划」筛选  
**THEN** 该菜谱出现在结果中。

**场景 S4.2**  
**GIVEN** 用户为菜谱选择指定日期且日期落在当前自然周内  
**WHEN** 用户打开「本周计划」筛选  
**THEN** 该菜谱 **SHALL** 出现在结果中。

### REQ-5 本地持久化

在无后端的前提下，系统 **SHALL** 使用浏览器 `localStorage` 持久化菜谱与记录，刷新后数据 **SHALL** 仍然存在。
